Voice AI Agent: End-to-End Implementation Flow
==============================================

This document explains how the voice agent works, from the user clicking "Start" to the AI responding with voice.

1. TRIGGERING THE AGENT
-----------------------
File: c:\Sanjay\frontend-insurance\src\app\pages\dashboard\customer-dashboard.page.ts
- [Line 2265] function toggleVoiceMode(): Toggles the voice mode on. Calls sendGreeting().
- [Line 2274] function sendGreeting(): 
  - Adds a "Hi Sanjay..." greeting message to the chat.
  - Sets `isVoiceProcessing` to `true` (pauses the microphone while speaking).
  - Calls `speakResponse()` to play the greeting using browser text-to-speech.
- [Line 2298] function speakResponse(): Uses `window.speechSynthesis` to speak the greeting. When finished, it sets `isVoiceProcessing` back to `false`.

2. CAPTURING AUDIO (FRONTEND)
-----------------------------
File: c:\Sanjay\frontend-insurance\src\app\pages\dashboard\customer-components\voice-agent\voice-agent.ts
- [Line 57] ngOnChanges(): Detects when `isProcessing` becomes `false`.
- [Line 67] calls startListening(): Begins the microphone capture.
- [Line 89] navigator.mediaDevices.getUserMedia(): Requests microphone permission and starts the audio stream.
- [Line 98] MediaRecorder Setup: Initializes recording of audio chunks.
- [Line 151] calls detectSilence(): Uses `AudioContext` and `AnalyserNode` to monitor volume in real-time.
- [Line 198] Logic: Once speech is detected for > 800ms, it flags `hasSpeechContent = true`.
- [Line 219] Logic: If 1200ms of silence follows the speech, it calls `stopRecordingAndEmit()`.
- [Line 125] onstop handler: Creates an `audio/webm` Blob from captured chunks and emits the `audioCaptured` event.

3. SENDING TO BACKEND
---------------------
File: c:\Sanjay\frontend-insurance\src\app\pages\dashboard\customer-dashboard.page.ts

5. PLAYING THE RESPONSE (FRONTEND)
----------------------------------
File: c:\Sanjay\frontend-insurance\src\app\pages\dashboard\customer-dashboard.page.ts
- [Line 2398] Response Received: The frontend gets the transcript, AI text, and the audio (as Base64 string).
- [Line 2407] updates chatMessages: Displays the conversation in the chat window.
- [Line 2415] calls playAudioBase64():

  - [Line 2456] audio.onended: Once the AI finishes speaking, `isVoiceProcessing` is set to `false`.
- The `VoiceAgent` component sees `isVoiceProcessing` go `false` and automatically restarts step 2 (Listening) for the next user question.

SUMMARY OF EXTERNAL SERVICES USED:
- Deepgram: Speech-to-Text (STT) - Fast and accurate.
- n8n: AI Orchestration - Handles logic and knowledge about policies.
- ElevenLabs: Text-to-Speech (TTS) - For the high-quality human voice.
