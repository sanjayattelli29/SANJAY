## 2. Azure Deployment Steps (Backend)

### Phase 1: Azure Resources
1. **Azure SQL Database**: Create a database named `AcciSureDB`.
2. **Azure App Service**: Create a web app for .NET 8.

### Phase 2: Configuration (Azure Portal -> App Service -> Environment Variables)
Set the following keys:
- `ConnectionStrings__DefaultConnection`: (Azure SQL Connection String)
- `AllowedOrigins__0`: `https://your-insurance-app.vercel.app` (Your Vercel URL)
- `JWT__SecretKey`: (Generate a long secure string)
- `JWT__Issuer`: `InsuranceAPI`
- `JWT__Audience`: `InsuranceClient`
- `ImageKit__PublicKey`: (From ImageKit)
- `ImageKit__PrivateKey`: (From ImageKit)
- `ImageKit__UrlEndpoint`: (From ImageKit)
- `ExternalApis__Deepgram__ApiKey`: ...
- `ExternalApis__Groq__ApiKey`: ...
- `ExternalApis__ElevenLabs__ApiKey`: ...

## 3. Vercel Deployment Steps (Frontend)

### Phase 1: Import Repository
1. Connect your Github/Gitlab/Bitbucket account.
2. Select the `frontend-insurance` project.

### Phase 2: Project Settings
1. **Framework Preset**: Angular.
2. **Build Command**: `ng build` (standard).
3. **Environment Variables**:
   - `GOOGLE_VISION_API_KEY`: (Your key)
   - Ensure the `environment.prod.ts` has the correct `apiUrl` pointing to your Azure App Service.
